ProjectTrack UI Design System + runtime reference

Documentacion del sistema visual de ProjectTrack Chrome


Esta guia reune el stack actual de Chrome: Bootstrap local, tema ProjectTrack, runtime full-tab, componentes editables y patrones reales del workspace.
La meta es que la documentacion se sienta como una referencia de framework y al mismo tiempo sirva como mapa exacto del HTML/CSS/JS que ya usa ProjectTrack.


Introduccion

ProjectTrack UI documenta el runtime Chrome actual: tokens, layout primitives, acciones, formularios, estados, superficies, navbar global, canal de releases y vistas full-tab. La entrada principal de producto es Chrome/workspace.html; la referencia visual Bootstrap-first quedo integrada en el workspace y en esta guia consultable.

Como convencion de desarrollo, el sistema adopta naming compatible con Bootstrap 5.3 para utilities y layout, pero mantiene breakpoints propios: sm = 550px, md = 700px, lg = 960px, xl = 1200px y xxl = 1400px.

Elemento actual Archivo Uso
Runtime principal Chrome/workspace.html Carga Bootstrap local, tema, full-tab, workspace CSS, Bootstrap bundle y src/main.js.
Navbar editable Chrome/components/global-navbar.html Template HTML con placeholders para marca, usuario, menu global y notices.
Capa workspace Chrome/styles/projecttrack-workspace.css Ajusta padding global, navbar, login wide, heroes y compatibilidad con estilos legacy.
Capa full-tab Chrome/styles/projecttrack-fulltab.css Define botones pastel/coral, user menu, cards web, hero full-tab y metric cards.
Guia consultable Chrome/docs/projecttrack-ui.html Copia interna que se abre desde el menu UI Guide de la extension.

Grid

El grid de ProjectTrack UI mezcla dos estrategias: una reticula explicita de 12 columnas para composiciones controladas y grids automaticos para cards que solo deben envolver contenido sin microajustes manuales.

Criterio de layout

.row + .col-* es la API publica recomendada de Bootstrap-ProjectTrack para documentacion, nuevas pantallas y refactors de layout simples. .pt-row + .pt-col-* se conserva para composiciones internas existentes o casos que dependen de su implementacion basada en CSS Grid. No mezcles ambas estrategias dentro del mismo bloque.

Columnas

Reticula de 12 columnas

.row crea la malla base. Cada hijo ocupa 12 columnas por defecto y luego se abre con .col-*, .col-sm-*, .col-md-*, .col-lg-*, .col-xl-* y .col-xxl-*.

NavBar

El navbar actual de ProjectTrack Chrome es un componente global editable. El HTML vive en Chrome/components/global-navbar.html y el runtime lo monta desde Chrome/src/projecttrack-app.js dentro de <nav class="navbar navbar-expand-lg bg-white border-bottom sticky-top pt-web-navbar pt-workspace-navbar">. Bootstrap controla el dropdown y ProjectTrack agrega la capa visual full-tab.

Template actual / preview

Esta es la estructura que se reutiliza en Dashboard, Projects, Project Details, Login, Profile y el resto del workspace.

Partes de la variante ProjectTrack
Chrome/components/global-navbar.html

Archivo fuente del template. Es el punto mas comodo para editar marca, acciones visibles y orden del menu sin tocar JavaScript.

.pt-workspace-navbar

Capa del navbar full-tab. Mantiene fondo blanco, borde inferior, posicion sticky y prioridad visual por encima del contenido.

.pt-workspace-brand + {{BRAND_MARK}}

Marca izquierda con icono, titulo y subtitulo alineados horizontalmente. El click navega a Dashboard.

.pt-workspace-navbar-actions

Zona derecha para acciones globales. Hoy contiene Refresh Data y el dropdown de usuario.

.pt-web-user-button + .pt-web-user-menu

Dropdown Bootstrap real con avatar, nombre, rol y accesos a Dashboard, Projects, Changes, Profile, UI Guide y Change History.

{{NOTICE}}

Placeholder donde renderNotice() inserta alertas globales de refresh, login, guardado o errores recuperables.

P / Small

El texto de cuerpo parte de step-0 como tamano base de 12px. Sobre esa base, Bootstrap-ProjectTrack expone una escala oficial para p, small y copy de apoyo.

La escala queda asi: step--3 = 3px, step--2 = 6px, step--1 = 9px, step-0 = 12px, step-1 = 18px, step-2 = 24px, step-3 = 36px, step-4 = 48px y step-5 = 72px.

H1 - H6

ProjectTrack UI no define una escala visual rigida para todos los encabezados HTML por igual. La jerarquia se resuelve por contexto de componente, asi que el tag semantico vive junto a una clase o wrapper que le da su tamano real en pantalla.

Ejemplos HTML H1 - H6

H1 - Titulo principal

Encabezado de mayor jerarquia semantica.

H2 - Titulo de seccion

Subdivide la pantalla o documento en bloques principales.

H3 - Titulo de subseccion

Nivel comun para panels, cards y vistas de detalle.

H4 - Titulo auxiliar

Funciona bien para agrupaciones secundarias.

H5 - Titulo menor

Util para bloques internos con poco peso visual.

H6 - Titulo minimo

Ultimo nivel semantico antes del texto de cuerpo.
