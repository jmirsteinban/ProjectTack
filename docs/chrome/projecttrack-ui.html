ProjectTrack UI Design System + runtime reference

Documentacion del sistema visual de ProjectTrack Chrome


Esta guia reune elementos compatibles con Bootstrap 5.3 y componentes reales del app.
La meta es que la documentacion se sienta como una referencia de framework y al mismo tiempo sirva como mapa exacto del CSS que ya usa ProjectTrack.


Introduccion

ProjectTrack UI documenta el microfrontend: tokens, layout primitives, acciones, formularios, estados y superficies. La capa de layout dentro del sistema se llama ProjectTrack Grid, con viewport minimo de 360px y ancho de diseno optimo de 550px.

Como convencion de desarrollo, el sistema adopta naming compatible con Bootstrap 5.3 para utilities y layout, pero mantiene breakpoints propios: sm = 550px, md = 700px, lg = 960px, xl = 1200px y xxl = 1400px.

Contrato responsive de ProjectTrack UI

La diferencia principal frente a Bootstrap esta en el viewport. Debajo de 550px el shell se comporta como zoom controlado; desde 550px en adelante empieza el responsive horizontal real.

Grid

El grid de ProjectTrack UI mezcla dos estrategias: una reticula explicita de 12 columnas para composiciones controladas y grids automaticos para cards que solo deben envolver contenido sin microajustes manuales.

Criterio de layout

.row + .col-* es la API publica recomendada de Bootstrap-ProjectTrack para documentacion, nuevas pantallas y refactors de layout simples. .pt-row + .pt-col-* se conserva para composiciones internas existentes o casos que dependen de su implementacion basada en CSS Grid. No mezcles ambas estrategias dentro del mismo bloque.

Columnas

Reticula de 12 columnas

.row crea la malla base. Cada hijo ocupa 12 columnas por defecto y luego se abre con .col-*, .col-sm-*, .col-md-*, .col-lg-*, .col-xl-* y .col-xxl-*.

P / Small

El texto de cuerpo parte de step-0 como tamano base de 12px. Sobre esa base, Bootstrap-ProjectTrack expone una escala oficial para p, small y copy de apoyo.

H1 - H6

ProjectTrack UI no define una escala visual rigida para todos los encabezados HTML por igual. La jerarquia se resuelve por contexto de componente, asi que el tag semantico vive junto a una clase o wrapper que le da su tamano real en pantalla.

Encabezado de mayor jerarquia semantica.

H3 - Titulo de subseccion

Nivel comun para panels, cards y vistas de detalle.

H4 - Titulo auxiliar

H5 - Titulo menor

H6 - Titulo minimo

Ultimo nivel semantico antes del texto de cuerpo.
